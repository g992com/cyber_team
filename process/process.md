# 软件开发流程说明（人读）

本规范采用流程驱动的阶段划分，便于 Agent 与人工按阶段执行并启用对应角色与 skill。

## 阶段顺序

0. **项目启动** — 根据用户问题/需求，按活动规范对项目活动进行裁剪，确定项目流程与计划；产出项目计划、流程裁剪说明、阶段与活动清单（可选：项目章程摘要）。依据 PMBOK 的按上下文裁剪及 ISO/IEC/IEEE 16326 的项目管理计划实践。
1. **需求** — 需求 elicitation、分析与优先级；产出用户故事、PRD、验收标准、路线图/Backlog。
2. **需求评审** — 对需求/用户故事/PRD 做完整性、一致性、可测性评审。
3. **架构/设计** — 技术选型、模块划分、接口与数据模型；产出架构设计、API 设计、数据库设计、ADR。
4. **设计/架构评审** — 对架构与设计文档、安全设计、DBA/DevOps 方案等评审。
5. **开发** — 前端/后端/全栈实现；可含 DBA 的 schema/迁移、安全的实现与配置。
6. **代码评审** — 对 PR/代码变更做规范、安全、性能、可维护性评审。
7. **测试** — 测试策略、自动化与手工测试、质量门禁。
8. **测试评审** — 对测试计划/用例的覆盖度与可执行性评审。
9. **部署/发布** — CI/CD、环境与发布策略、回滚预案。
10. **运维/复盘** — 可靠性、容量与成本、事件响应与改进。

评审阶段可与主阶段并行或在其后（如需求评审在需求阶段末，代码评审在开发 PR 提交时）。

## 阶段转换规则

何时进入下一阶段（验收满足、用户确认、产出物就绪等）由项目/团队在落地时约定，约定后可在本目录或 README 中写明。

阶段定义与顺序的机器可读版本见 `phases.yaml`。
